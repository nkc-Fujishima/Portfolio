タイトル : TANK FRONT LINE3---------------------------------------------------------------------------------------
ジャンル : 3Dシューティングアクション
開発時期、期間 : 2023/2～2023/5  3～4か月程度
開発人数 : 3人
開発環境、ツール、言語、ライブラリ ： Unity、C#、UniRx
担当箇所 : チームリーダー、プレイヤー、ボス敵、ステージセレクト、企画立案、動画制作
制作背景 : 
本作は日本ゲーム大賞アマチュア部門2023に向けて制作したゲームとなります。
日本ゲーム大賞アマチュア部門2023のテーマが"こだわり"ということで昔触ったゲームの操作性を取り入れて爽快感のあるゲームに仕上げたいと思い、これまで触れてきたゲームの要素を取り入れ完成させたゲームとなります。

こだわり : 
ゲーム部分は塊魂のような操作感でアーマードコアのオートロックシステムを採用し、地球防衛軍みたいに多数の敵を相手にしていくという様々なゲームの要素を掛け合わせていくことにこだわりました。
結果として、初心者には触りにくいゲームとなってしまいましたが、自身が体現したいものが作りきれたという実感がもてるゲームとなりました。

プログラム部分では、内積や外積をUnity既存のメソッドに頼らずに計算式を調べてソースコードに実装したこと、またenumで簡素なステート切り替えを実装したことです。
前者においては内積や外積などのメソッドがサポートされていない言語を触れることになった場合、すぐに対応出来るようにと思い、勉強しました。

後者においては、ボス戦のAIを担当したので、内積での角度計算をもとに行動を変えたいと思い、ステート切り替えの機構を作ろうと思いました。

アピールポイント : 
特殊な操作性をもとに爽快なジェット移動をしながら敵を撃つ点を楽しんで頂きたいと思います。
また敵を殲滅させるだけでなく、拠点の防衛、特定地点まで移動、ボス敵撃破など単調にならないように工夫をしております。簡単なステージ説明も実装しておりますので、ミッション目的の確認がてら見て頂けるとうれしく思います。

大変だった点 : 
チームリーダー兼プログラマとして動いていたので、メンバーの持つ疑問点などの解消を行いつつゲームの根幹部分のプログラムの実装を手早く終わらせることは不可能だと考え、ステージのクリア条件や雑魚敵等のプログラムはチームメンバーに任せ、自身はすぐに完成しなくても問題ないステージセレクト画面などの細かい作業を受け持つことでトラブルが発生したときに自身の仕事で手が離せないという状況を無くしました。

また、プレイヤープログラムを担当していましたが、こちらはプロジェクトを発足し仕様決めとメンバーに役割を振る段階までにある程度の基本操作を実装することで他のメンバーがプレイヤーがいなくて実験出来ないという状況を無くしました。

タイトル : SHI KA KU----------------------------------------------------------------------------------------------
ジャンル : 3Dステルスパズルアクション
開発時期、期間 : 2024/9～2024/11 2か月程度
開発人数 : 2人
開発環境、ツール、言語、ライブラリ ： Unity、C#、UniRx、UniTask、DoTween
担当箇所 : チームリーダー、エネミー、ステージセレクト、一部モーション制作、一部シェーダー制作、企画立案、動画制作
制作背景 : 
「四角いやつらの死角から刺客を送り込め！」

テーマ「しかく」という言葉から出てきた言葉を組み合わせて出来たキャッチコピーから制作に至ったゲームです。
敵の視線を誘導して、敵の死角から動物を送り込んで敵を倒していくゲームとなります。

ゲーム全体の見た目を緩め、かつポップチックな色合いにすることでゲーム自体の印象を残しやすくしています。

こだわり : 
テーマの「しかく」を活用したいため、世界観や使う文字のフォントなどは角ばったものを使うことで、「四角」な世界観を表現しました。

次にゲーム性として、「死角」を使うことで、敵に見られている場合は倒すことが出来ない、そのため何かを用いて視線を誘導し、敵の死角から何かしらを働きかけようという運びになりました。

最後にプレイヤーが撃つ弾を「刺客」として扱い、視線誘導した敵の背後から攻撃してステージをクリアしていくゲームにしようと思い、「しかく」という言葉をテーマにしたステルスパズルアクションゲームが完成しました。

アピールポイント : 
アピールポイントとして、「TANK FRONT LINE3」では各ステージでマップをあらかじめ生成していたためゲームの実行ファイルが大きくなってしまったという点がありました。
そのため、今回はステージ生成情報を記録し、ステージ開始時に生成を行うようにすることで実行ファイルの大きさを大幅に削減することに成功しました。

また、今作までの間にデザインパターン等の作り方を学んだため、プログラム制作における時間の効率化を図ることができました。

モーションに関して、使用したアセットには歩く等に基本動作のみだったため、攻撃や、やられモーション等はプログラム制作の合間を縫って自身で制作しております。

大変だった点 : 
お借りしたアセット内に使いたいモーションが存在しませんでした。ただ人型のモーションであれば自身である程度まかなえるため世界観に合わせた動きを制作し、ゲーム内に実装しました。

タイトル : Rescue!!AMBOT!!---------------------------------------------------------------------------------------
ジャンル : 2Dパズル
開発時期、期間 : 2021/10～2021/12  2か月程度
開発人数 : 6人
開発環境、ツール、言語、ライブラリ ： Unity、C#
担当箇所 : チームリーダー、プレイヤープログラム
制作背景 : 
学校生活初めてのチーム制作で作ったゲームです。
まずは、チームで共同で作るという経験を積むため、分かりやすいゲームを
真似てみようということで作り始めたゲームです。

こだわり : 
絵を描ける方がチームにいたので、アセットに頼らずにビジュアル面を強化しました。
背景はビル群のパターンを4つほど作って頂き、置く位置のパターンを変えることで
書き直す手間を省きました。

大変だった点 : 
学校生活初めての共同制作だったので、進捗確認等が難しい点が目立ちました。
ただ、制作期間内は朝礼、終礼、Teamsに毎日の進捗報告を上げることで
少し管理しやすくなりました。

タイトル : WASDタイピングゲーム-----------------------------------------------------------------------------------
ジャンル : クイズ
開発時期、期間 : 2023/6～2023/6  1週間程度
開発人数 : 1人
開発環境、ツール、言語、ライブラリ ： Unity、C#
制作背景 : 
授業においてゲームジャムを行う期間がありましたので、
担任の決めるテーマ「あける」をもとにこれまで作ったことの無い
クイズ系のものを作ろうと思い、本作を仕上げました。

こだわり : 
この制作の前にボス敵のAIを作っており、ステート管理を
してみたいと考え、Switch文でステート管理を行っています。

大変だった点 : 
制作終盤において動くものを作ることを優先してしまい、
依存関係やステージ生成がおざなりになってしまった。
リファクタリングで改善しました。
